# HousePuzzle 开发环境配置说明

本文档说明如何在本地搭建 **HousePuzzle** 项目的 Unreal Engine 开发环境，并完成首次编译与运行。

## 项目概览

| 项目 | 说明 |
|------|------|
| 项目名称 | HousePuzzle |
| 引擎版本 | Unreal Engine **5.7.4** |
| 项目类型 | C++ 第一人称解谜游戏 |
| 目标平台 | Windows（Win64） |
| 图形 API | DirectX 12（Shader Model 6） |
| C++ 标准 | C++20 |
| 默认启动地图 | `/Game/Map/MyScene` |

项目基于 UE5 第一人称模板扩展而来，包含卧室、客厅、书房、阳台等房间谜题逻辑，并保留了 Horror / Shooter 等变体玩法代码。

---

## 硬件建议

本项目启用了 **Lumen**、**Nanite**、**硬件光追** 等特性，对显卡有一定要求。

| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| 操作系统 | Windows 10 64 位（19041+） | Windows 11 64 位 |
| CPU | 6 核以上 | 8 核及以上 |
| 内存 | 16 GB | 32 GB 及以上 |
| 显卡 | 支持 DX12 的独立显卡 | RTX 系列或同等级别（支持硬件光追更佳） |
| 磁盘空间 | 约 150 GB（含引擎与缓存） | SSD，预留 200 GB 以上 |

## 软件依赖

### 1. Unreal Engine 5.7

本项目在 `HousePuzzle.uproject` 中指定引擎关联版本为 **5.7**。

**推荐安装路径（与当前开发机一致）：**

```
C:\Soft\UE_5.7
```

若使用 Epic Games Launcher 安装，请确保安装 **5.7.x** 分支，且小版本不低于 **5.7.4**。

> 当前机器已验证的引擎版本：`5.7.4`（Changelist: 51494982）

### 2. Visual Studio 2022

UE 5.7 在 Windows 上编译 C++ 代码需要 **Visual Studio 2022**（不支持 VS 2019）。

| 要求项 | 版本 |
|--------|------|
| Visual Studio | **17.8** 或更高 |
| MSVC 工具集 | **v14.38** 或更高（推荐 v14.44 / VS 17.14.x） |
| Windows SDK | **10.0.22621**（Windows 11 SDK 22H2） |

**必须安装的工作负载与组件：**

- 使用 C++ 的游戏开发（`Microsoft.VisualStudio.Workload.NativeGame`）
- 使用 C++ 的桌面开发（`Microsoft.VisualStudio.Workload.NativeDesktop`）
- .NET 桌面开发（`Microsoft.VisualStudio.Workload.ManagedDesktop`）
- Windows 11 SDK (10.0.22621.0)
- MSVC v143 - VS 2022 C++ x64/x86 生成工具

项目根目录已提供 `.vsconfig`，可在 Visual Studio Installer 中通过 **“导入配置”** 一键安装推荐组件：

```
HousePuzzle/.vsconfig
```

**建议额外安装（Unreal 开发体验更好）：**

- `Component.Unreal.Ide` — Unreal Engine IDE 集成
- `Component.Unreal.Debugger` — Unreal Engine 调试器

### 3. 其他工具（可选）

| 工具 | 用途 |
|------|------|
| [Git](https://git-scm.com/) | 版本管理 |
| [Epic Games Launcher](https://store.epicgames.com/) | 安装与管理 UE 引擎 |
| Cursor / Visual Studio / Rider | 代码编辑与调试 |

## 环境搭建步骤

### 步骤 1：获取项目代码

将项目克隆或复制到本地目录，例如：

```
C:\Projects\UEProjects\HousePuzzle
```

### 步骤 2：关联 Unreal Engine

1. 双击 `HousePuzzle.uproject`
2. 若弹出引擎版本选择窗口，选择已安装的 **Unreal Engine 5.7**
3. 若引擎安装路径非默认位置，可在 Epic Games Launcher → 库 → 引擎版本 → 下拉菜单 → **管理引擎安装** 中确认路径

也可手动编辑注册表或通过右键 `.uproject` → **Switch Unreal Engine version** 完成关联。

### 步骤 3：生成 IDE 工程文件

首次打开或修改 `Build.cs` / 插件配置后，需要重新生成 Visual Studio 解决方案：

**方式 A：通过编辑器**

右键 `HousePuzzle.uproject` → **Generate Visual Studio project files**

**方式 B：通过命令行**

```powershell
& "C:\Soft\UE_5.7\Engine\Build\BatchFiles\Build.bat" -projectfiles -project="C:\Projects\UEProjects\HousePuzzle\HousePuzzle.uproject" -game -rocket -progress
```

> 请将引擎路径替换为你本机实际的 UE 5.7 安装目录。

生成完成后，项目根目录会出现 `HousePuzzle.sln`。

### 步骤 4：编译项目

**方式 A：通过 Unreal Editor（推荐首次使用）**

1. 双击 `HousePuzzle.uproject` 打开编辑器
2. 若提示缺少模块，点击 **Yes** 自动编译
3. 等待编译完成，编辑器将自动启动

**方式 B：通过 Visual Studio**

1. 用 Visual Studio 2022 打开 `HousePuzzle.sln`
2. 将启动项目设为 `HousePuzzle`
3. 配置选择 `Development Editor` + `Win64`
4. 菜单 **生成 → 生成解决方案**（或 `Ctrl+Shift+B`）
5. 编译成功后，在 VS 中按 `F5` 启动调试，或直接打开 `.uproject`

**方式 C：通过命令行**

```powershell
& "C:\Soft\UE_5.7\Engine\Build\BatchFiles\Build.bat" HousePuzzleEditor Win64 Development -Project="C:\Projects\UEProjects\HousePuzzle\HousePuzzle.uproject" -WaitMutex
```

### 步骤 5：验证运行

编译成功后：

1. 启动 Unreal Editor
2. 确认默认地图 `MyScene` 能正常加载
3. 点击 **播放（PIE）** 验证游戏逻辑

## 项目目录结构

```
HousePuzzle/
├── HousePuzzle.uproject      # 项目描述文件（引擎版本、模块、插件）
├── HousePuzzle.sln           # Visual Studio 解决方案（生成后产生）
├── .vsconfig                 # VS 推荐组件配置
├── Config/                   # 项目配置（引擎、输入、游戏设置等）
├── Content/                  # 美术资源、蓝图、地图等
├── Source/                   # C++ 源代码
│   ├── HousePuzzle/          # 主游戏模块
│   ├── HousePuzzle.Target.cs
│   └── HousePuzzleEditor.Target.cs
├── Binaries/                 # 编译产物（自动生成，勿手动编辑）
├── Intermediate/             # 中间构建文件（自动生成）
├── Saved/                    # 日志、本地配置、缓存（自动生成）
└── DerivedDataCache/         # DDC 缓存（自动生成）
```

### C++ 源码模块划分

`Source/HousePuzzle/` 下主要目录：

| 目录 | 说明 |
|------|------|
| `Core/` | 游戏实例等核心逻辑 |
| `Interaction/` | 交互组件与可交互基类 |
| `Puzzle/` | 谜题管理器、通用谜题类型 |
| `Puzzles/Bedroom` | 卧室谜题 |
| `Puzzles/LivingRoom` | 客厅谜题（幻灯机、拼图板等） |
| `Puzzles/Study` | 书房谜题 |
| `Puzzles/Balcony` | 阳台谜题（风铃等） |
| `UI/` | UMG 相关 Widget |
| `Misc/` | 出口、结局触发等 |
| `Variant_Horror/` | 恐怖变体玩法 |
| `Variant_Shooter/` | 射击变体玩法（含 AI、武器） |

## 日常开发工作流

### 修改 C++ 代码后

- **小改动**：保存文件后回到 Editor，按 `Ctrl+Alt+F11` 触发 Live Coding，或点击编译按钮
- **大改动**（修改头文件、新增类、改 Build.cs）：关闭 Editor，在 VS 中重新编译，再启动 Editor

### 修改蓝图 / 资源

直接在 Unreal Editor 的 Content Browser 中编辑，保存即可。

### 查看日志

运行时日志位于：

```
Saved/Logs/HousePuzzle.log
```

C++ 代码中可使用日志分类 `LogHousePuzzle` 输出调试信息。

## 常见问题排查

### 1. 提示找不到引擎 / 引擎版本不匹配

- 确认已安装 UE 5.7
- 右键 `.uproject` → **Switch Unreal Engine version** → 选择 5.7
- 检查 `HousePuzzle.uproject` 中 `"EngineAssociation": "5.7"` 是否正确

### 2. 编译失败：找不到 Windows SDK

在 Visual Studio Installer 中安装 **Windows 11 SDK (10.0.22621.0)**。

### 3. 编译失败：MSVC 版本不兼容

- 升级 Visual Studio 2022 至 **17.8+**
- 安装推荐工具集 **v14.44**（VS 17.14.x）
- 避免使用 UE 5.7 已知有问题的 MSVC 版本（14.39.x、14.40.x 部分版本）

### 4. 提示缺少模块 `HousePuzzle`

删除以下目录后重新编译：

```
Binaries/
Intermediate/
```

然后重新生成工程文件并编译。

### 5. Shader 编译慢 / 首次打开耗时长

属于正常现象。`DerivedDataCache/` 会缓存着色器编译结果，后续启动会快很多。建议将项目和 DDC 放在 SSD 上。

### 6. Live Coding 失败

关闭 Editor，在 Visual Studio 中执行完整编译后重新打开。
