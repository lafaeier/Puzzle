# 错误记录与解法

每个错误只记一次，包含现象、根因、解法。避免团队重复踩坑。

---

## 编译器类

### E1：Live Coding 阻止命令行编译

**现象**：运行 Build.bat 报错 `Unable to build while Live Coding is active`

**根因**：UE 编辑器打开时，Live Coding 会锁定编译产物。命令行编译和 Live Coding 互斥。

**解法**：
1. 完全关闭 UE 编辑器
2. 若仍有问题，用任务管理器强制结束 `UnrealEditor.exe` 进程
3. 再执行 Build.bat

**预防**：每次命令行编译前先关编辑器。

---

### E2：增量编译 vs 全量重编

**现象**：第一次编译 104 秒，后续仅修改 .cpp 时 13 秒。修改 .h 时编译时间明显更长（触发了更多重编译单元）。

**根因**：UBT 只重编译变更的文件。`.h` 修改会影响所有 `#include` 它的文件。

**解法**：无，这是正常行为。

**预防**：尽量减少 `.h` 的修改频率。新增函数优先在 `.cpp` 里实现，需要暴露给蓝图时才在 `.h` 声明。

## 配置类

### E3：GameInstance Class 未设置，日志无输出

**现象**：PIE 运行时 Output Log 搜不到 `LogPuzzle` 或 `RoomGameInstance`

**根因**：引擎默认使用 `UGameInstance` 基类，不会自动使用自定义子类。必须在 Project Settings 中显式指定。

**解法**：
1. 编辑器顶部菜单：编辑 → 项目设置
2. 左侧：项目 → 地图和模式
3. 右侧找到 Game Instance Class，下拉选择 `RoomGameInstance`
4. 关闭窗口（自动保存）

**预防**：任何继承自引擎基类的新类（GameInstance、GameMode、PlayerController、Character）都需要在 Project Settings 中指定或在 GameMode 构造函数中设置，否则引擎不会使用。

## 编辑器操作类

### E4：关卡蓝图菜单灰显

**现象**：顶部菜单 "蓝图 → 打开关卡蓝图" 是灰色的，无法点击

**根因**：没有关卡加载到视口中。关卡蓝图是跟具体关卡绑定的，不加载关卡就没有蓝图可打开。

**解法**：
1. 底部内容浏览器（Content Browser）中找到关卡文件（如 `Content/FirstPerson/Lvl_FirstPerson`）
2. 双击打开
3. 视口顶部标签页显示关卡名称后，"打开关卡蓝图" 即可点击

**预防**：操作关卡蓝图前先确认视口中已加载关卡。

## 蓝图连线类

### E5：纯函数节点无白色执行引脚

**现象**：从 Event BeginPlay 的白色箭头拖线到 "获取游戏实例"（Get Game Instance），连不上

**根因**：UE 中有两种节点：
- **执行节点**：有白色箭头（执行输入/输出），参与执行流，如 `Mark Cabinet Opened`、`Print String`、`Cast To`
- **纯函数节点**：只有数据引脚（圆形），不参与执行流，如 `Get Game Instance`、数学运算、取值操作

纯函数的输出值在"被需要时"自动计算，不需要执行流触发。

**解法**：
- 纯函数不接入白线链路
- 纯函数的 Return Value → 拖数据线（不是白线）→ 连到其他节点的数据引脚

**识别方法**：右键搜索后，节点标题旁边没有白色三角形图标的就是纯函数。

---

### E6：Cast To 节点被修剪（执行引脚未连接）

**现象**：编译蓝图时报错 "Cast To RoomGameInstance 已被修剪，因为其执行引脚未连接，连接的值不可用"

**根因**：`Cast To`（转换为）节点有执行引脚。它需要白色执行线穿过它。如果只有数据引脚被连接而执行引脚未连接，编译时节点会被丢弃。

**正确连线**：
```
Event BeginPlay ──白线──→ Cast To RoomGameInstance ──白线──→ Mark Cabinet Opened
                              ↑ Object                    ↑ Target
                          Get Game Instance        Cast 节点的 As Room Game Instance
                          的 Return Value          连到 Mark 节点的 Target
```

**解法**：Cast To 节点的执行引脚（左上角白色箭头）必须连在白线链路中。

## 蓝图连线类 — 正确模式速查

### 模式一：调用 GameInstance 上的函数

```
Event BeginPlay ─白线→ CastToRoomGameInstance ─白线→ 目标函数 ─白线→ PrintString
                          ↑ Object                    ↑ Target
                      GetGameInstance             Cast的As输出
```

### 模式二：读取 GameInstance 的 UPROPERTY 变量

```
Event BeginPlay ─白线→ CastToRoomGameInstance ─白线→ PrintString
                          ↑ Object                    ↑ In String
                      GetGameInstance             Cast的As输出 → 拖线 → 选变量名
```

### 模式三：创建 UObject（SaveGame）

```
Event BeginPlay ─白线→ CreateSaveGameObject ─白线→ PrintString
                          Class: 下拉选XXX
```
