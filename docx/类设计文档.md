# 类设计文档

## 类比：Java电商 vs UE游戏

```
Java电商项目                          UE游戏项目
─────────────────────────────────────────────────────────
数据库表 (CREATE TABLE)          ←→  UPROPERTY 成员变量（定义"存什么"）
Entity类 (User.java, Order.java) ←→  UObject/Actor子类（定义"是什么"）
DAO层 (UserMapper.java)          ←→  GameInstance（全局数据读写，跨"页面"不丢）
Service层 (OrderService.java)    ←→  各Actor的内部逻辑（具体做什么）
Controller层 (OrderController)   ←→  接口 IInteractable（统一入口，分发请求）
Spring注入 (@Autowired)          ←→  UE反射/查找（GetGameInstance, FindComponent）
Session/Cookie                   ←→  GameInstance（跨请求/跨关卡保持状态）
前端页面 (JSP/Thymeleaf)         ←→  关卡 (.umap) + 蓝图子类（展示层）
```

关键差异：
- Java的表结构靠SQL定义，UE的数据结构靠 `UPROPERTY()` 宏 + C++成员变量定义。UE没有独立DDL。
- Java靠Spring管理对象生命周期，UE靠引擎自身的GC和反射系统管理。
- Java的"跨页面数据"靠Session/Redis，UE的"跨关卡数据"靠GameInstance。

## 一、数据层（= 数据库表设计）

### 1.1 URoomGameInstance — 全局数据容器

```
类比：Java中一个全局可访问的HashMap<String, Object>，存所有跨页面的session数据
```

```cpp
UCLASS()
class URoomGameInstance : public UGameInstance
{
    GENERATED_BODY()

public:
    // ===== 主线状态 =====
    // 卧室储物柜是否已打开
    UPROPERTY(BlueprintReadOnly, Category = "进度")
    bool bCabinetOpened = false;

    // 是否已获得客厅钥匙
    UPROPERTY(BlueprintReadOnly, Category = "进度")
    bool bHasKeyToLivingRoom = false;

    // ===== 支线状态（暂不实现）=====
    UPROPERTY(BlueprintReadOnly, Category = "进度")
    bool bHasBulb = false;

    UPROPERTY(BlueprintReadOnly, Category = "进度")
    bool bLightIsOn = false;

    UPROPERTY(BlueprintReadOnly, Category = "进度")
    bool bDrawerUnlocked = false;

    // ===== 背包数据 =====
    // 持有物品的ID列表
    UPROPERTY(BlueprintReadOnly, Category = "背包")
    TArray<FName> InventoryItemIDs;
};
```

```
对应Java概念：
  - bCabinetOpened、bHasKeyToLivingRoom 就是数据库表的字段
  - 每个字段存一个游戏进度的"事实"
  - 关卡切换 = Java里页面跳转，但session不丢
```

### 1.2 URoomSaveGame — 存档结构

```
类比：Java中把session数据序列化成JSON写入文件/Redis，下次登录恢复
```

```cpp
UCLASS()
class URoomSaveGame : public USaveGame
{
    GENERATED_BODY()

public:
    UPROPERTY()
    bool bCabinetOpened = false;

    UPROPERTY()
    bool bHasKeyToLivingRoom = false;

    UPROPERTY()
    bool bHasBulb = false;

    UPROPERTY()
    bool bLightIsOn = false;

    UPROPERTY()
    bool bDrawerUnlocked = false;

    UPROPERTY()
    TArray<FName> InventoryItemIDs;

    UPROPERTY()
    FName CurrentLevel;
};
```

```
与GameInstance的关系：
  GameInstance = 运行时内存中的状态（关了就没了）
  SaveGame     = 磁盘上的持久化副本（下次打开还在）
  
  保存：GameInstance → 拷贝字段 → SaveGame → 写磁盘
  加载：磁盘 → SaveGame → 拷贝字段 → GameInstance
```

## 二、交互接口（= Controller层统一入口）

### 2.1 IInteractable — 交互契约

```
类比：Java中的接口，所有"可处理请求"的类都实现它
     Spring的 @RequestMapping —— 统一入口，分发到不同Handler
```

```cpp
UINTERFACE(BlueprintType)
class UInteractable : public UInterface
{
    GENERATED_BODY()
};

class IInteractable
{
    GENERATED_BODY()

public:
    // 核心方法：被F键触发时执行的行为
    // 类比：Controller的 @PostMapping("/interact")
    virtual void Interact_Implementation(APlayerCharacter* Instigator) = 0;

    // 获取交互提示文字（准星对准时显示）
    // 类比：Controller返回给前端的提示信息
    virtual FString GetInteractPrompt() const = 0;
};
```

```
调用流程类比：
  玩家按F = 用户在浏览器点击按钮
  LineTrace = Nginx路由匹配URL
  IInteractable = Controller接口
  Interact_Implementation() = Controller的具体方法
```

## 四、交互物层（= Service层具体实现）

### 4.1 AStorageCabinet — 储物柜 ★主线

```
类比：Java中的 OrderService.createOrder() —— 核心业务逻辑
      UPDATE game_progress SET cabinet_opened = 1 WHERE ...
      INSERT INTO inventory (item_id, player_id) VALUES ('key', ...)
```

```cpp
UCLASS()
class AStorageCabinet : public AActor, public IInteractable
{
    GENERATED_BODY()

public:
    // ===== 配置（蓝图设置）=====
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "配置")
    TSubclassOf<class APickupItem> SpawnItemClass;  // 生成什么道具

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "配置")
    FTransform SpawnTransform;                       // 道具生成位置

    // ===== 状态 =====
    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "状态")
    bool bOpened = false;

    // ===== 实现接口 =====
    virtual void Interact_Implementation(APlayerCharacter* Instigator) override;
    virtual FString GetInteractPrompt() const override;
};
```

```
Interact_Implementation() 内部逻辑（伪代码）：

void AStorageCabinet::Interact_Implementation(APlayerCharacter* Instigator)
{
    // 1. 幂等校验：已打开则直接返回
    if (bOpened) return;

    // 2. 获取全局数据（类比：获取session）
    URoomGameInstance* GI = GetGameInstance<URoomGameInstance>();

    // 3. 执行业务：生成道具（类比：INSERT）
    GetWorld()->SpawnActor<APickupItem>(SpawnItemClass, SpawnTransform);

    // 4. 更新状态：自己 + 全局（类比：UPDATE）
    bOpened = true;
    GI->bCabinetOpened = true;

    // 5. 播放表现：开门动画、音效（由蓝图侧处理）
}
```

### 4.2 APickupItem — 可拾取道具

```
类比：商品对象，加入购物车 = 加入背包，下单 = 写入GameInstance
      INSERT INTO inventory VALUES (...)
      DELETE FROM scene_actors WHERE id = ...（从场景中移除）
```

```cpp
UCLASS()
class APickupItem : public AActor, public IInteractable
{
    GENERATED_BODY()

public:
    // ===== 配置（蓝图设置）=====
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "配置")
    FName ItemID;           // 唯一标识，如 "KeyToLivingRoom"

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "配置")
    FText ItemName;         // 显示名，如 "客厅钥匙"

    // ===== 实现接口 =====
    virtual void Interact_Implementation(APlayerCharacter* Instigator) override;
    virtual FString GetInteractPrompt() const override;
};
```

```
Interact_Implementation() 内部逻辑：

void APickupItem::Interact_Implementation(APlayerCharacter* Instigator)
{
    // 1. 加入背包组件（类比：add to cart）
    UInventoryComponent* Inv = Instigator->GetComponentByClass<UInventoryComponent>();
    if (Inv) Inv->AddItem(ItemID);

    // 2. 更新全局数据（类比：write to session）
    URoomGameInstance* GI = GetGameInstance<URoomGameInstance>();
    if (ItemID == "KeyToLivingRoom") GI->bHasKeyToLivingRoom = true;

    // 3. 从场景中移除（类比：delete from DOM）
    Destroy();
}
```

### 4.3 ADoorTrigger — 门触发器

```
类比：Java的权限校验中间件
      if (session.hasKey("KeyToLivingRoom")) {
          redirect("/living_room");
      } else {
          return "需要钥匙才能进入";
      }
```

```cpp
UCLASS()
class ADoorTrigger : public AActor, public IInteractable
{
    GENERATED_BODY()

public:
    // ===== 配置（蓝图设置）=====
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "配置")
    FName RequiredKeyID;           // 需要的钥匙ID，如 "KeyToLivingRoom"

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "配置")
    FName TargetLevel;             // 目标关卡，如 "Level_LivingRoom"

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "配置")
    bool bRequiresKey = true;      // 是否需要钥匙（返程设为false）

    // 提示文字
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "配置")
    FString SuccessPrompt;         // "按F开门"

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "配置")
    FString FailPrompt;            // "需要客厅钥匙"

    // ===== 实现接口 =====
    virtual void Interact_Implementation(APlayerCharacter* Instigator) override;
    virtual FString GetInteractPrompt() const override;
};
```

```
Interact_Implementation() 内部逻辑：

void ADoorTrigger::Interact_Implementation(APlayerCharacter* Instigator)
{
    URoomGameInstance* GI = GetGameInstance<URoomGameInstance>();

    // 1. 权限校验
    bool bCanPass = !bRequiresKey || GI->InventoryItemIDs.Contains(RequiredKeyID);

    if (bCanPass) {
        // 2. 切换关卡
        UGameplayStatics::OpenLevel(this, TargetLevel);
    } else {
        // 3. 显示提示
        APlayerController* PC = Instigator->GetController<APlayerController>();
        // 在屏幕上显示 FailPrompt
    }
}
```

## 六、类之间的关系总图

```
                    ARoomGameModeBase
                          │ 指定默认类
          ┌───────────────┼───────────────┐
          ↓               ↓               ↓
  ARoomPlayerController  ARoomPlayerCharacter  URoomGameInstance
  (管理输入+UI)          (第一人称+F键)       (全局状态，跨关卡)
          │               │   │               ↑
          │               │   │ 持有           │ 读写
          │               ↓   │               │
          │      UInventoryComponent          │
          │      (背包数据)                    │
          │                                   │
          │  OnInteract() ─LineTrace→ 命中Actor是否实现IInteractable?
          │                                      │
          │              ┌───────────┬───────────┤
          │              ↓           ↓           ↓
          │     AStorageCabinet  APickupItem  ADoorTrigger
          │     (打开→生成钥匙)  (拾取→入背包) (校验→切关卡)
          │              │           │           │
          └──────────────┴───────────┴───────────┘
                 各自实现 Interact_Implementation()
                 各自读写 URoomGameInstance 更新全局进度
```

## 七、开发顺序（被依赖的先写）

```
第1步: URoomGameInstance + URoomSaveGame + ARoomGameModeBase
       理由：无外部依赖，后面的类全部依赖它们

第2步: IInteractable + UInventoryComponent
       理由：接口是契约，组件是工具，无其他游戏类依赖

第3步: ARoomPlayerCharacter + ARoomPlayerController
       理由：依赖第2步的接口和组件

第4步: AStorageCabinet + APickupItem
       理由：依赖第1步的GameInstance + 第2步的接口 + 第3步的角色

第5步: ADoorTrigger
       理由：依赖第1步的GameInstance（读状态）+ 第2步的接口

已完成主线。第6步（支线）按需追加。
```
